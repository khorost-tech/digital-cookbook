# zig/hello-comptime

Минимальный запускаемый пример к статье [«Zig: место среди C, Rust и C++»](https://khorost.tech/zig/zig-positioning-among-c-rust-cpp/): hello, дженерик через `comptime` и встроенный в язык тест.

Проверено на **Zig 0.16** (`zig run` / `zig test` / `zig build test` — зелёные). Zig до 1.0 — на другой версии API стандартной библиотеки и системы сборки могут отличаться.

## Файлы

- `main.zig` — код: `main` + comptime-дженерик `max` + блок `test`.
- `build.zig` — система сборки на самом Zig (шаги `run` и `test`).

## Запуск

Проще всего — без системы сборки, напрямую из компилятора:

```bash
zig run main.zig       # собрать и запустить
zig test main.zig      # прогнать тесты (std.testing.allocator ловит утечки)
```

Через `build.zig`:

```bash
zig build run          # собрать и запустить
zig build test         # прогнать тесты
```

## Что показывает пример

- **comptime вместо макросов/шаблонов:** `max` принимает тип `T` как `comptime`-параметр — одна функция работает для `i32`, `f64` и любого другого типа, а разрешение происходит на этапе компиляции.
- **Тесты как часть языка:** блок `test` живёт рядом с кодом, запускается тем же тулчейном.
